Complete Thinking Process (Interview Cheat Sheet)

Whenever you get an LLD problem:
Read requirements carefully.
Identify entities (classes).
Identify actions (methods/services).
Divide into modules.
Look for places where design patterns fit:
Multiple algorithms → Strategy
Multiple object types → Factory (optional) or inheritance
One-to-many notifications → Observer
Complex communication → Mediator
Create interfaces wherever multiple implementations are expected.
Separate responsibilities into services (SRP).
Draw a rough class diagram.
Define the execution flow.
Start coding from models → interfaces → implementations → services → main program.
